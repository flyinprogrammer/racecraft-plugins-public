# Roadmap - Map of Content

Navigation map for the cross-platform plugin runtime release-blocker lane.

## Epics (curated)

### Runtime Contract

Why: Stop guessing whether Windows support is documentation, environment setup,
or architecture work. Inventory the real active surfaces and choose one runtime
contract before implementation starts.

- XPLAT-001 Runtime Inventory and Architecture Contract

### Runner Foundation

Why: Create one structured helper invocation path before porting behavior. The
runner owns path handling, JSON envelopes, subprocess safety, and platform
preflight so every later helper is consistent.

- XPLAT-002 Cross-Platform Runner Foundation

### Helper Migration

Why: Port behavior in reviewable batches, proving parity before any active Claude
or Codex surface changes.

- XPLAT-003 Read-Only Helper Port
- XPLAT-004 Mutation, Install, and PR-Emission Helper Port

### Public Release Gate

Why: Public release needs proof that the installed plugin works on native
Windows, macOS, and Linux without Unix-shell workarounds.

- XPLAT-005 Claude/Codex Cutover and Native Windows Release Gate
