# Contract: migrate-structure.sh

## Command

```bash
speckit-pro/skills/speckit-autopilot/scripts/migrate-structure.sh (--dry-run|--apply) [--repo-root <path>]
```

Exactly one mode flag is required.

## Purpose

Run the repository-level structure migration. This creates or verifies the v1
repo structure marker and drives Tier-0 generated navigation backfill through
`generate-spec-index.sh`.

## Inputs

| Input | Required | Rules |
|-------|----------|-------|
| `--dry-run` | one mode required | Read-only, allowed on dirty trees. |
| `--apply` | one mode required | Requires clean tree before backup or mutation. |
| `--repo-root <path>` | optional | Defaults to the repository containing the plugin script. Must be a directory inside a git worktree. |

## Output

Stdout emits one compact JSON report matching
[migration-report-json.md](./migration-report-json.md). The report includes:

- pending or current `.specify/structure-version.json` state
- active-feature state
- skipped frozen/in-flight specs
- Tier-0 roadmap-MOC backfill decisions
- generated index update decisions
- planned or created backup path
- no-op current-state items

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Successful dry-run, successful apply, or safe no-op. |
| `2` | Usage error, invalid repo root, invalid active-feature state in apply, dirty-tree apply block, backup failure, marker write failure, or generator failure. |

## Dry-Run Behavior

- Does not create, edit, move, stamp, or regenerate files.
- Allowed on dirty trees.
- Reports the planned backup path without creating it.
- Reports pending Tier-1 marker work when `.specify/structure-version.json` is
  absent or below v1.
- Reports no-op when the marker is current and generated navigation is current.
- Reports invalid active-feature state but does not mutate.

## Apply Behavior

1. Resolve repo root.
2. Parse active-feature state.
3. Detect marker state, generated update decisions, and no-op current state
   without mutating files.
4. Exit before backup or mutation if active-feature state is invalid.
5. Run dirty-tree check.
6. Exit before backup or mutation if the tree is dirty.
7. If marker and generated navigation are already current, emit a no-op report
   with `backup.created` false and do not create a backup.
8. Create forced backup outside the repo and record the path.
9. Create or update `.specify/structure-version.json` to:

   ```json
   {"structureVersion":1}
   ```

10. Invoke or delegate to `generate-spec-index.sh` so Tier-0 roadmap-MOC INDEX
   rendering is regenerated by the single generated-zone writer.
11. Emit the final JSON report.

## Tier-0 Rules

- Tier-0 includes completed or archived historical specs only when the repo
  marker is integer `structureVersion >= 1`.
- Tier-0 never stamps, moves, or renames completed historical spec files.
- Tier-0 excludes the valid in-flight spec from `.specify/feature.json`.
- Tier-0 applies candidate eligibility before ID joins or roadmap row rendering.
  Non-SpecKit alpha namespaces and date-first legacy namespaces are reported as
  `skipped_out_of_scope` with stable reasons and do not emit INDEX rows.
- Tier-0 selects durable link targets in this order:
  1. `../../../specs/<dir>/SPEC-MOC.md` for version-gated specs.
  2. `../../../specs/<dir>/spec.md` for legacy on-disk specs without a gated MOC.
  3. `../../../.specify/memory/spec.md#<section-slug>` for archive-memory-only specs.

## Generator Integration

`migrate-structure.sh` must not patch generated INDEX rows directly. It creates
or verifies the state needed by `generate-spec-index.sh`, then delegates
generated-zone updates to that script.

## Idempotency

After successful repository migration, dry-run and apply report no-op current
state. No backup is created and no marker write or generated-zone update occurs
on a clean already-current repository.
