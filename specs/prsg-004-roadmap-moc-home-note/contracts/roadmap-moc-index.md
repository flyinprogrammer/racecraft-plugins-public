# Contract: Roadmap-MOC GENERATED INDEX zone — FR-011…FR-019, FR-022

The authoritative byte-level contract for the **roadmap-MOC home note's** GENERATED INDEX
zone, produced by the activated `render_index()` in the single shared
`generate-spec-index.sh`. It is the roadmap-MOC sibling of PRSG-003's
`prs-manifest.schema.md` / `sentinel-grammar.md` (which remain authoritative for the
per-spec-MOC zones — unchanged by this spec). The generator matches this contract
byte-for-byte; the Layer-4 home-note fixture pins it.

`bash` + `jq` only; no network, no `gh` — a pure function of committed files (SC-004 / SC-008).

## Scope (context-scoping — FR-011 / FR-018)

This contract governs the INDEX zone **only when the target is the roadmap-MOC home note**.
`render_index` is already invoked against every per-spec SPEC-MOC's present INDEX zone, where
it MUST continue to render **empty** (link-free) — emitting rows there would break the pinned
PRSG-003 byte-for-byte contracts (SC-005). The activation branches on a target-type signal
originated at discovery in `main()` and threaded into `render_index` as a defaulted
parameter (empty output is the default → the spec-MOC path stays byte-identical).

## Home-note location & discovery

```text
docs/ai/specs/<slug>-roadmap-MOC.md
```

- Discovered by `main()` via the filename glob `docs/ai/specs/*-roadmap-MOC.md` (0..N
  matches), each gated by `moc_is_gated` (the `structureVersion` bare-integer gate in
  `lib/moc-frontmatter.sh`).
- Processed **in addition to and disjoint from** the repo-root `specs/*/SPEC-MOC.md` scan;
  the two trees do not overlap. Zero home notes → clean no-op (consistent with the
  no-`specs/`-directory no-op, SC-012).
- A non-gated / legacy / fenceless home note is **skipped**, not an error (total/safe reads).

## INDEX zone source rows (the repo-wide scan)

The INDEX is **repo-wide** over every gated spec under `<REPO_ROOT>/specs/` (PRSG-004 assumes
a single roadmap; per-roadmap scoping for coexisting roadmaps is deferred to PRSG-011). For
each `specs/<dir>/SPEC-MOC.md`:

| Field | Source | Notes |
|-------|--------|-------|
| `spec_id` | that SPEC-MOC's frontmatter `spec_id`, via `moc_frontmatter_field` | the visible link text; also normalized via `moc_normalize` for ordering |
| `status` | that SPEC-MOC's frontmatter `status`, via `moc_frontmatter_field` | the at-a-glance signal; MAY be empty/missing (still rendered, FR-015) |
| `<dir>` | the spec directory basename | used to build the relative link target |

Only version-marked (`moc_is_gated`) SPEC-MOCs are indexed; legacy/non-marked SPEC-MOCs are
**skipped** (FR-016), consistent with the existing gating.

## Row format (exact)

Each in-scope spec renders as **exactly one line**:

```text
- [<spec_id>](../../../specs/<dir>/SPEC-MOC.md) · <status>
```

- The separator is the **U+00B7 MIDDLE DOT** — bytes `0xC2 0xB7` — framed as **space, 0xC2
  0xB7, space** (`<sep>`), reusing the `PRS_SEP` convention so it is byte-consistent with the
  PRS renderer. NOT an ASCII dot, NOT a link.
- The link is a relative `[]()` Markdown target resolving `../../../specs/<dir>/SPEC-MOC.md`
  (home note in `docs/ai/specs/`, spec tree in repo-root `specs/`) — **never** a
  `[[wikilink]]` (FR-014 / SC-006).
- The INDEX contains **link + status only** — no table, no dashboard, no H1 title parse
  (FR-022); the row content is a pure function of frontmatter fields.

## Ordering (FR-013)

Rows are ordered **normalized-ID ascending**, using `moc_normalize` on each `spec_id` as the
sort key. The visible link text is the raw `spec_id`; only the sort key is normalized. The
repo-root directory scan is itself `LC_ALL=C sort`-ed so enumeration order never leaks across
machines (SC-009).

## Empty / missing status (FR-015) — pin the exact bytes

A SPEC-MOC with an empty or missing `status` MUST still produce a row (link + blank status);
the row MUST NOT be dropped. The row is rendered with the separator present and the status
field empty (matching the `render_prs` precedent of always emitting the field separator):

```text
- [<spec_id>](../../../specs/<dir>/SPEC-MOC.md) · 
```

> **Determinism note (SC-004 idempotence):** whichever exact byte sequence the implementation
> emits for the empty-status case (including any trailing space after the separator) is FROZEN
> by the Layer-4 home-note fixture so a second run is a zero-byte diff. The fixture MUST
> include a spec whose `status` is empty/missing to lock this byte form. Trailing whitespace on
> a line is the classic idempotence breaker; the fixture exists to catch any later trim/no-trim
> drift.

## Zone framing & assembly (reused from PRSG-003 — `sentinel-grammar.md`)

- The INDEX zone is bounded by the **exact** sentinel pair (full-line byte equality):

  ```text
  <!-- GENERATED:INDEX:START (do not edit; regenerated by generate-spec-index.sh) -->
  <!-- GENERATED:INDEX:END -->
  ```

- The home note carries **only the INDEX sentinel pair** (FR-002) — NOT the PRS or BACKLINKS
  pairs. With `st_index=present, st_prs=absent, st_backlinks=absent`, the generator takes the
  **whole-zone-rewrite** path (`_rewrite_present_zones`), filling INDEX and leaving the rest
  untouched; the inject-if-missing path does NOT fire (it requires all three pairs absent).
  No `prs.json` is needed for the home note.
- These sentinel bytes are SHIPPED in the PRSG-002 `roadmap-moc-template.md` (the shape
  `speckit-prd` emits from), so they are a committed, greppable artifact rather than
  AI-authored prose. They MUST byte-match the generator's `INDEX_START` / `INDEX_END`
  constants exactly, or the generator silently never fills the INDEX.
- Whole-zone replace: the entire body between the sentinels is regenerated every run
  (staleness-free).

## Empty / absent behavior

| Condition | Behavior |
|-----------|----------|
| Home note present, `specs/` has ≥1 gated spec | INDEX filled with one row per gated spec, normalized-ID ascending |
| Home note present, `specs/` empty or all-legacy | INDEX rendered **empty** (link-free) — NOT an error |
| No home note (glob matches 0) | clean no-op; the `specs/` scan proceeds unaffected |
| Home note not gated (no `structureVersion`) | skipped — NOT an error |
| Home note gated but MISSING its INDEX sentinel pair (all three GENERATED pairs absent) | **fail-safe exit 2, no write** (FR-017a) — a gated home note without its INDEX zone is malformed; the generator MUST NOT take the inject-if-missing path (which would inject all three zones and render PRS/BACKLINKS against `docs/ai/specs/`, contradicting FR-002) |
| Home note's INDEX sentinel pair unbalanced (orphan/duplicated START or END) | **fail-safe exit 2, no write** — inherited from the shared marker-balance check (`_zone_state`/`err`), same as the spec-MOC path |
| `status` empty/missing on a spec | row still emitted (link + blank status) — see "Empty / missing status" |
| `spec_id` empty/missing on a gated spec | **row SKIPPED** (no row) — `spec_id` is the link text AND sort key; an absent/empty `spec_id` in a gated marker is a PRSG-002 lint violation, so it is not fabricated into a zero-width `[]()` row (FR-015a). NOT symmetric with empty `status`. |
| A spec-MOC target (NOT the home note) | INDEX zone rendered **empty** — the spec-MOC path is byte-identical to pre-activation (SC-005) |

## Determinism & idempotence (SC-004 / FR-019)

- Pure function of committed files: identical committed inputs → byte-identical INDEX zone.
- A second consecutive write run over the same committed SPEC-MOCs yields a **zero-byte
  diff** (the home note is folded into the same PASS-1/PASS-2 diff + atomic-write machinery as
  the spec-MOCs, so `--check` staleness and idempotence cover it with no new code).
- Every per-spec SPEC-MOC.md remains **byte-identical** to its pre-activation output (the
  spec-MOC INDEX still renders empty), so the pinned PRSG-003 contracts pass unchanged.

## Worked example

`specs/` contains three gated spec dirs (`prsg-001-foo`, `prsg-002-bar`, `prsg-010-baz`) and
one legacy non-gated dir (`legacy-thing`). Their SPEC-MOC frontmatter:

- `prsg-001-foo` → `spec_id: PRSG-001`, `status: complete`
- `prsg-010-baz` → `spec_id: PRSG-010`, `status: in-progress`
- `prsg-002-bar` → `spec_id: PRSG-002`, `status:` (empty)
- `legacy-thing` → no `structureVersion` (skipped)

The home note `docs/ai/specs/myproject-roadmap-MOC.md` INDEX zone regenerates to:

```text
<!-- GENERATED:INDEX:START (do not edit; regenerated by generate-spec-index.sh) -->
- [PRSG-001](../../../specs/prsg-001-foo/SPEC-MOC.md) · complete
- [PRSG-002](../../../specs/prsg-002-bar/SPEC-MOC.md) · 
- [PRSG-010](../../../specs/prsg-010-baz/SPEC-MOC.md) · in-progress
<!-- GENERATED:INDEX:END -->
```

(Ordered normalized-ID ascending: `prsg 001` < `prsg 002` < `prsg 010`. `PRSG-002` renders
with a blank status — link present, status empty, row not dropped. `legacy-thing` is absent —
not gated. The `·` is U+00B7. Every per-spec SPEC-MOC's own INDEX zone stays empty.)
